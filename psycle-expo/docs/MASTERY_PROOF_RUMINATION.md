# Mastery Proof: Rumination

## Summary

このドキュメントは、`反芻` という 1 テーマだけで

- Core
- Mastery
- Return
- Refresh

をどこまで重複感なく切れるかを検証するための proof である。

目的は `本当に量が足りるか` を机上ではなく構造として確認すること。

---

## Theme

- `theme_id`: `rumination`
- recurring pain:
  - 夜に同じことを考え続けて眠れない
  - 会話後に「あれでよかったか」を何度も繰り返す
  - 失敗の場面が頭から離れない

---

## Core Unit

Core は `何が起きているか -> どう戻るか` の地図を作る本編。

### Core 1. 気づく
- `lesson_job`: 反芻が始まっていることに気づく
- `target_shift`: 「考えている」から「反芻しているかも」に変わる
- `takeaway_action`: いま頭の中で同じループが起きているか確認する

### Core 2. 見分ける
- `lesson_job`: 整理と反芻を見分ける
- `target_shift`: 「整理しているつもり」から「これは整理ではない」に変わる
- `takeaway_action`: 同じ結論に戻っているかを見る

### Core 3. 理解する
- `lesson_job`: 反芻が情動維持になりやすいと理解する
- `target_shift`: 「役に立つ思考」から「疲弊を増やすループ」に変わる
- `takeaway_action`: 続けるほど良くなるとは限らないと覚える

### Core 4. 証拠を見る
- `lesson_job`: 自分の反芻の証拠を並べる
- `target_shift`: 「全部事実」から「事実と推測は混ざる」に変わる
- `takeaway_action`: 事実 / 解釈 / 予測を分ける

### Core 5. その場で戻る
- `lesson_job`: 反芻から抜ける最小行動を選ぶ
- `target_shift`: 「考え切る」から「戻る」を選べるようになる
- `takeaway_action`: 1分以内の切替行動を1つ選ぶ

### Core 6. 再発時に戻る
- `lesson_job`: 再発しても戻れる
- `target_shift`: 「また失敗した」から「戻れる」に変わる
- `takeaway_action`: 戻り方の問いを1つ持つ

---

## Mastery Set

Mastery は `同テーマを別価値で返す` ための変奏。

### Mastery 1. 夜の反芻
- `lesson_job`: 寝る前の反芻に早く気づく
- `difference`: `scene_change`

### Mastery 2. 会話後の反芻
- `lesson_job`: 会話後の自己反省ループを見分ける
- `difference`: `scene_change`

### Mastery 3. 仕事後の反芻
- `lesson_job`: 仕事のミスを引きずる反芻を扱う
- `difference`: `scene_change`

### Mastery 4. SNS後の反芻
- `lesson_job`: 比較と反芻が混ざる場面を切り分ける
- `difference`: `scene_change`

### Mastery 5. “考え続けた方がいい” を疑う
- `lesson_job`: 反芻を正当化する判断を疑う
- `difference`: `judgment_change`

### Mastery 6. “今結論を出さない” を選ぶ
- `lesson_job`: 今ここで結論を出さない選択を学ぶ
- `difference`: `judgment_change`

### Mastery 7. 反芻の再発初期に戻る
- `lesson_job`: 反芻の初期サインで切り替える
- `difference`: `relapse_context`

### Mastery 8. 反芻後の自己否定を減らす
- `lesson_job`: 反芻した自分への二次ダメージを減らす
- `difference`: `relapse_context`

### Mastery 9. 現実行動に戻す
- `lesson_job`: 反芻後に次の具体行動へ戻す
- `difference`: `transfer_context`

### Mastery 10. 他テーマへの接続
- `lesson_job`: 反芻と不安回避 / 先延ばしの接続を見抜く
- `difference`: `transfer_context`

結論:
- `反芻` だけで Mastery は **10本** までは十分切れる
- それ以上は quality drift が起きやすいので、以降は Refresh / Replay 側で回す方がよい

---

## Return Set

Return は短い再入場。新知識を入れない。

### Return 1. いま反芻か確認する
- `lesson_job`: 落ち着く / 思い出す / 再開する
- `question_count`: `3`

### Return 2. 事実と予測を分ける
- `lesson_job`: 再開の前に混線をほどく
- `question_count`: `4`

### Return 3. 次の1分を選ぶ
- `lesson_job`: その場で戻る
- `question_count`: `5`

---

## Refresh Candidates

Refresh は新研究や新 evidence で既存 lesson を更新する。

### Refresh A. explanation_update
- `parent`: Core 3
- 新しい review で、反芻の説明がよりよくなった

### Refresh B. intervention_update
- `parent`: Core 5
- 既存の切替行動より、負荷が低く再現しやすい介入が見つかった

### Refresh C. scene_update
- `parent`: Mastery 4
- SNS 後の反芻に関する具体 scene を追加した

### Refresh D. evidence_grade_update
- `parent`: Core 2
- 単発研究ベースだった説明を review ベースへ昇格した

---

## Capacity Estimate

- Core: `6`
- Mastery: `10`
- Return: `3`
- Refresh candidates: `4+`

1テーマあたりの実用供給量の初期見積もり:
- **固定本編**: `6`
- **重複感なく返せる変奏**: `8〜10`
- **短い再入場**: `2〜3`
- **研究更新で差し替え可能な枠**: `4以上`

合計すると、`反芻` だけで **20本前後の実用スロット** を持てる。

---

## Decision

- `1テーマ = Core 6` を基本にする
- `Mastery 8〜10` を目標レンジにする
- `Return 2〜3` を標準にする
- `Refresh` は lesson 数を増やすより、既存 slot の更新を優先する

この proof で確認できること:
- `Core だけ` では量は足りない
- しかし `Core + Mastery + Return + Refresh` なら、同テーマでもかなり長く回せる
- 量を作る鍵は `新テーマ追加` ではなく `Mastery の設計` である
